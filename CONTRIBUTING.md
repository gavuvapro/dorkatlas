# Contributing to DorkAtlas

Thank you for your interest in contributing to DorkAtlas!

## How to Contribute

1. **Fork the repository** and create your feature branch from `main`.
2. **Make your changes** following our guidelines.
3. **Ensure all safety rules** are respected — never add any prohibited search patterns.
4. Submit a pull request with a clear description.

## Code Guidelines

- Use TypeScript strictly
- Follow the existing component and file structure
- Keep all new operators educational and ethical
- All new features must pass the Safety Engine validation

## Reporting Issues

Use GitHub Issues and label appropriately (bug, enhancement, documentation).

## Code of Conduct

Please follow our [Code of Conduct](CODE_OF_CONDUCT.md).
